Step 1

Importing the 3D Models

Before building anything, we need to bring our 3D model assets into the Unity project. These models include the road segments, traffic cones, roadwork barriers, and the player character.

1

Download the Model Pack

Go to the provided link (in Brightspace) and download the 3D model pack as a .zip file.

2

Extract the Archive

Unzip the file on your computer to reveal the Models folder containing all assets.

3

Drag into Unity

Drag and drop the entire folder into the Unity Project panel. It appears under a "Models" folder with all road, obstacle, and player meshes.

4

Clean Up the Scene

Delete the old placeholder ground object from the hierarchy ‚Äî we'll replace everything with our new 3D tiles.

üí°

You'll see models named Road, TrafficCone, RoadworksBarrierHigh, RoadworksBarrierLow, and the Player character.

Step 2

Creating the Base Tile

The first tile is a clean road segment with no obstacles. This serves as the foundation ‚Äî every other tile variant is built on top of this base pattern.

1

Create an Empty Game Object

In the Hierarchy panel, right-click ‚Üí Create Empty. Rename it to "Tile1" and reset its Transform to origin (0, 0, 0).

2

Add the Road Model

From the Models folder, drag the Road model into Tile1 as a child object. This is the visual road surface.

3

Add a Box Collider

Select the Road child ‚Üí Add Component ‚Üí Box Collider. This prevents the player from falling through the road.

4

Create the Prefab

Create an Assets ‚Üí Prefabs folder. Drag Tile1 from the Hierarchy into this folder to save it as a reusable prefab.

Critical: Without the Box Collider on the road, your player will fall straight through the surface. Always add physics colliders to walkable geometry!

Step 3

Traffic Cone Obstacles

Now we create tile variants that force the player to dodge left or right. Traffic cones are placed on specific lanes, and the player must switch lanes to avoid them.

1

Unpack & Duplicate Tile1

Right-click the Tile1 prefab instance ‚Üí Unpack Prefab. Rename it to "Tile2". This breaks the prefab link so you can modify freely.

2

Add Traffic Cones

Drag the TrafficCone model into Tile2. Set position X to 2.5 (right lane), Y to 0.443 (above the road), and scale to 0.7 on all axes.

3

Add Box Collider to Cone

Each cone needs its own Box Collider for collision detection. This is how the game knows the player hit an obstacle.

4

Duplicate & Mirror

Press Ctrl+D to duplicate the cone. Move the copy to X = -2.5 (left lane). Now Tile2 has cones on both sides ‚Äî the player goes through the center.

Step 5

The Slide Obstacle

The high barrier introduces the slide mechanic. It's positioned above the player's normal running height, so they must duck underneath it. This adds vertical variety to the gameplay beyond just lane-switching.

1

Import RoadworksBarrierHigh

Drag the high barrier model from the Models folder. Adjust the scale factor to 0.7 via the model importer, then click Apply.

2

Position the Barrier

Move it slightly toward the bottom of the tile so it sits at a height the player must slide under. The exact position depends on your player height.

3

Configure the Box Collider

Add a Box Collider and use the Edit Collider button (the icon in the Inspector) to manually adjust the collider bounds. If the player hits this collider, they lose.

4

Save as Prefab (Tile 5)

Drag into the Prefabs folder. The slide obstacle is ready for level generation!

Step 6

The Jump Obstacle

The low barrier is the counterpart to the slide obstacle ‚Äî here, the player must jump over a ground-level barrier. The Box Collider is positioned low so the player clears it with a jump action.

1

Swap the Barrier Model

Unpack a tile prefab, delete the high barrier, and drag in the RoadworksBarrierLow model from the Models folder.

2

Position & Scale

Place it near ground level. Use the Scale Tool (R key) to resize visually, or type precise values in the Inspector.

3

Fine-Tune the Box Collider

Add a Box Collider and use the Edit Collider icon to shape it precisely. The player must jump over this ‚Äî if their feet touch the collider, they lose.

4

Test the Jump

Press Play and hit the Up Arrow key. Verify the player can clear the barrier. Adjust collider or jump force if needed.

‚ö†Ô∏è

Collider positioning is critical! If the collider is too tall, even jumping won't save the player. If too small, they can walk right through. Use the Edit Collider tool to get it just right.

Step 7

Assembling & Testing the Level

Now comes the fun part ‚Äî snapping tiles together and playtesting! Each tile is 30 units long along the Z-axis, so you simply place them end-to-end with a Z-offset of 30 per tile.

1

Place Tiles End-to-End

Drag prefabs into the scene. Set Z positions to 0, 30, 60, 90‚Ä¶ etc. Each tile length is 30 units, so they align seamlessly.

2

Mix Tile Variants

Alternate between clean road, cone tiles, slide barriers, and jump barriers to create interesting gameplay sequences.

3

Hit Play & Test

Press Play and run through the level. Test all obstacle types: dodge cones, slide under barriers, jump over barriers.

üí°

Remember: In the next tutorial, these tiles will be spawned procedurally at runtime ‚Äî so you won't need to manually place them. This is just for visual testing.
