Introduction to C# in Unity

Getting started with Visual Studio

Welcome to programming in C# for Unity! We'll be using Visual Studio as our code editor. The key thing to understand right now is the two main methods in every Unity script: Start() and Update().
Video Transcript

"Welcome to programming in C# for Unity Hub. I'm using Visual Studio for that reason. I will take you very slow and we will write in C#, the programming language. The only thing to worry about right now is these two methods: the Start method and the Update method."

Fig 1: Introduction to C# scripting in Unity

Start() Method

Runs once as soon as the script is enabled. Perfect for initialization code.

Update() Method

Runs constantly every single frame while the script is enabled.

02

Accessing GameObject Properties

Changing the name via script

In the Start method, we can access our GameObject and its properties. By typing gameObject.name, we can access and modify the name that appears in the Inspector. This demonstrates how scripts interact with the Unity Editor.
Video Transcript

"In the Start we can type gameObject... and then we can access the name of the object. We have many options to choose from - all the options are properties of the inspector: the static, the tag, the layer, and the name."

BirdScript.cs
void Start() { // Access and change the GameObject's name gameObject.name = "Bob Burington"; }
Fig 2: Changing GameObject name via script
Important: Save Your Files!

Always save your script (File → Save or Ctrl+S) before going back to Unity. Changes won't apply until saved!

03

Understanding References

Connecting scripts to components

By default, scripts can only talk to the GameObject's name and Transform. To access other components like Rigidbody2D, we need to create a reference - a special slot that connects our script to the component.
Video Transcript

"Initially script can only talk to the gameObject name and the Transform. Right now the script is completely unaware of the other components. So we need to make a special slot on this script for Rigidbody2D so we can then talk to it and send commands. This is called a reference."

Fig 3: Understanding the concept of references
What's a Reference?

Think of a reference as a "connection wire" between your script and a component. Without it, your script can't communicate with that component.

04

Creating a Public Reference

Declaring the Rigidbody2D variable

We declare a public variable of type Rigidbody2D. The "public" modifier makes this variable appear in the Inspector, where we can drag and drop the Rigidbody2D component to connect it.
Video Transcript

"We're going to declare a variable or reference and this reference should be public and it's called myRigidBody. This public modifier is going to allow this myRigidBody reference to appear on the Inspector side."

BirdScript.cs
public class BirdScript : MonoBehaviour { // Public reference - appears in Inspector public Rigidbody2D myRigidBody; void Start() { } }
Fig 4: Creating a public Rigidbody2D reference
Connecting the Reference

After saving the script, go to Unity, select the Bird, and drag the Rigidbody2D component into the "My Rigid Body" field in the Inspector.

05

Understanding Vector2

2D coordinates and movement

Vector2 represents 2D coordinates with X and Y values. The center is (0, 0). Moving along X changes horizontal position, moving along Y changes vertical position. For upward movement, we use (0, 1) - no horizontal change, positive vertical change.
Video Transcript

"Vector2 is two dimensions. At the center is (0, 0). Once we move, the X axis and Y axis coordinates will change depending on the direction of our GameObject. Since we want just to move up our bird, the zero is going to be for X and one for Y."

Fig 5: Understanding Vector2 coordinates
Vector2(1, 0) → Move Right Positive X, no Y change
Vector2(-1, 0) → Move Left Negative X, no Y change
Vector2(0, 1) → Move Up No X change, positive Y
Vector2(0, -1) → Move Down No X change, negative Y
06

Setting Velocity

Making the bird fly continuously

Using our Rigidbody2D reference, we can set the velocity property. Velocity controls how fast and in which direction the object moves. We multiply Vector2.up by a value to control speed.
Video Transcript

"We're going to look for velocity... and fix this velocity to Vector2. We can control the movement of the bird by specifying a multiplier that controls the scale of the movement. In this case we'll multiply it by 10."

BirdScript.cs
void Update() { // Set upward velocity - runs every frame! myRigidBody.velocity = Vector2.up * 10; }
Fig 6: Setting velocity to make the bird fly
Problem: Continuous Flight!

With this code in Update(), the bird flies up constantly! There's no gravity effect. We need to add a condition to control when this happens.

07

Introduction to If Statements

The fundamental gate of programming

An if statement is like a gate. You can surround code with a "fence" (curly brackets), and every frame that code will be ignored UNLESS the game meets the specified condition - then the gate opens and the code executes.
Video Transcript

"It's time to use the most fundamental bit of programming: the if statement. An if statement is like a gate. You can surround some code with a fence (curly brackets). Every frame that code will be completely ignored unless the game meets the specified condition written on the gate - in which case the gate opens and the code is read and executed."

Fig 7: Understanding if statements as gates
The Gate Analogy

Think of curly brackets { } as fences around your code. The condition in parentheses ( ) is the key to the gate. Only when the condition is true does the gate open!

08

Detecting Space Bar Input

Using Unity's Input system

To detect when the player presses the space bar, we use Unity's Input system. Input.GetKeyDown(KeyCode.Space) asks Unity: "Was the space bar pressed on this frame?" We check if this equals true.
Video Transcript

"This time we are not talking to a component, we are actually talking to Unity itself - specifically its Input system. We will write Input, then GetKeyDown, and in brackets KeyCode.Space. This asks Unity if the space bar has been pressed on this frame."

BirdScript.cs
void Update() { // Check if space bar was pressed this frame if (Input.GetKeyDown(KeyCode.Space) == true) { // Flap! Add upward velocity myRigidBody.velocity = Vector2.up * 10; } }
Fig 8: Detecting space bar input
Single vs Double Equals

= (single) assigns a value. == (double) checks if two things are equal. In conditions, always use ==!

09

Testing the Flap Mechanic

Your bird now responds to input!

Now in Update, every frame the game checks: "Has the space bar been pressed?" If yes, the code fires and the bird flaps upward. If not, it skips the code and tries again next frame. We've created a character that reacts to player input!
Video Transcript

"Now in Update, every frame the game will go to the gate and be asked: 'Has the space bar just been pressed?' If yes, the code will fire and the bird will flap. If not, it will skip the code in the curly brackets and try again next frame. Tada! The bird goes up when we press space!"

Fig 9: Testing the flap mechanic in action
Success!

You've created a character that responds to player input! The bird falls with gravity and flaps up when you press space - the core mechanic of Flappy Bird!

10

Creating Flap Strength Variable

The smart way to adjust values

Changing the multiplier (10) in the script every time you want to test different speeds is tedious. The smart solution: create a public float variable called flapStrength and use it instead of the hardcoded number.
Video Transcript

"Instead of changing the multiplier from 10 to 6, going back to Unity, playing... it's not a really smart way. The best way is to declare a public reference, a float, and name it flapStrength. This variable is going to replace the value 10 in our script."

BirdScript.cs
public class BirdScript : MonoBehaviour { public Rigidbody2D myRigidBody; public float flapStrength; // Appears in Inspector! void Update() { if (Input.GetKeyDown(KeyCode.Space)) { myRigidBody.velocity = Vector2.up * flapStrength; } } }
Fig 10: Creating the flapStrength variable
11

Adjusting Values in the Inspector

No more script editing for tweaks!

Now the flapStrength variable appears in the Inspector! You can change it from 0 to 10 (or any value), play the game, and instantly see the results. No need to go back to the script - this is the professional way to tune game values.
Video Transcript

"If we go back to Unity you will see on the Inspector side that a component flapStrength is added, initialized to zero. We're going to change it to 10 and play the game. No need to go back to script and change it there! We can play with this flapStrength component, increase it, decrease it, and you will see the velocity of our bird without worrying about changing the script."

Fig 11: Adjusting flapStrength in the Inspector
