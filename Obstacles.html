The Secret of Flappy Bird

The illusion of flight revealed

Here's the secret: while it looks like the bird is flying through pipes, the bird is actually staying completely still! The pipes move across the screen from right to left, creating the illusion of movement.
Video Transcript

"The Secret of a flying bird with Flappy Bird is while it looks like the bird is flying, it's actually flapping along through a wall of pipes. The bird is not actually flying - the bird stays completely still and the pipes move across the screen."

Fig 1: The secret behind Flappy Bird's movement
GAME DESIGN TRICK

This is a common technique in 2D games! Instead of moving the player through a level, move the level past the player. It's simpler and more efficient.

PARENT-CHILD BENEFITS

When you move, rotate, or scale a parent object, all children move with it. The parent acts as a pivot point for the entire group!

04

Adding Components to the Pipe

Sprite Renderer and Box Collider

Just like the bird, add a Sprite Renderer for the pipe image. But this time, add a Box Collider 2D (not Circle!) because pipes are rectangular. We don't need Rigidbody2D since pipes aren't affected by physics.
Video Transcript

"Let's repeat what we did for the bird - add Sprite Renderer for the pipe image and add a collider, a Box Collider 2D this time and not Circle Collider. And we don't need a Rigidbody because it is not going to be affected by physics."

Fig 4: Adding Sprite Renderer and Box Collider 2D
Sprite Renderer Makes the pipe visible by displaying the pipe image
Box Collider 2D Rectangular collision shape - perfect for pipes!
No Rigidbody2D Needed Pipes don't need gravity - they move via script only

06

Creating the Bottom Pipe

Duplicate and flip

Duplicate the Top Pipe object (Ctrl+D), rename it "Bottom Pipe", and flip it upside down by changing the Y Scale to -1. Then move it below the bird. Now you have a complete pipe pair!
Video Transcript

"Finally we can duplicate this whole Top Pipe object, call it Bottom Pipe and flip it upside down by changing the Y scale to minus 1. Then move it down below the bird."

Fig 6: Duplicating and flipping to create bottom pipe
FLIP TRICK: Y SCALE = -1

Setting Y Scale to -1 flips the sprite vertically. This is much easier than rotating 180 degrees or creating a separate upside-down image!

09

Vector3 vs Vector2

Why we need three dimensions

For movement, we use Vector3 instead of Vector2 because Transform has three coordinates (X, Y, Z). Even in 2D games, Unity is fundamentally a 3D engine, so Transform.position requires Vector3.
Video Transcript

"This time we're going to have to use a Vector3 instead of a Vector2 because the Transform has three numbers - X, Y, and Z. Even though we are making our game in 2D, Unity is still fundamentally a 3D engine."

VECTOR2 (X, Y)

Used for Rigidbody2D.velocity - physics calculations only need 2D coordinates.

VECTOR3 (X, Y, Z)

Used for Transform.position - Unity's 3D engine requires all three axes.

Fig 9: Understanding Vector3 for Transform movement
10

Writing the Movement Code

Making pipes move left

In Update, we take the current position and add to it using Vector3.left * moveSpeed. This moves the object left every frame. The pipes now smoothly glide across the screen!
Video Transcript

"We will take the current transform.position and then equals - don't forget the sign equals - and we want to add to its current position. So write transform.position again (that means current position), plus Vector3.left times moveSpeed. That way the game object will move on the left side."

PipeMoveScript.cs
public float moveSpeed = 10; void Update() { // Move left every frame // current position = current position + left direction * speed transform.position = transform.position + (Vector3.left * moveSpeed); }
Fig 10: Pipes moving smoothly across the screen!
