Start Method

Calculating the Offset

In the Start() method, we capture the initial distance between the camera and the player. This offset stays constant throughout the game.

{
    offset = transform.position - target.position;
}
üì∑

transform.position

The camera's position in world space (since this script is attached to the Main Camera).

üèÉ

target.position

The player's position in world space. We subtract it to get the relative distance between camera and player.

üì¶

Example

Camera at (0, 8, -10), Player at (0, 0, 0) ‚Üí offset = (0, 8, -10). The camera is 8 units up and 10 units behind.

Camera Logic

Calculating the New Position

Each frame, we build a new position vector. The X and Y stay locked to the camera's current values, while the Z follows the player plus the original offset.

Position Calculation 
Vector3 newPosition = new Vector3(
    transform.position.x,   // X stays the same
    transform.position.y,   // Y stays the same
    offset.z + target.position.z  // Z follows player
);

transform.position = newPosition;
üîí X & Y Locked

We use the camera's own transform.position.x and .y so it doesn't drift sideways or vertically when the player moves.

üèÉ Z Follows

offset.z + target.position.z means: wherever the player is on Z, place the camera at that position plus the original gap (e.g., -10 units behind).

If you put camera follow logic in Update(), the camera might read the player's old position (before it moved this frame), causing jitter.

Smoothness

Adding Smooth Follow with Lerp

Instead of snapping instantly, we use Vector3.Lerp to smoothly blend between the camera's current position and the target position each frame.

Without Smoothing 
// Instant snap ‚Äî no smoothness
transform.position = newPosition;
With Smoothing ‚ú® 
// Smooth interpolation
transform.position = Vector3.Lerp(
    transform.position,
    newPosition,
    10 * Time.deltaTime
);
üîÑ Vector3.Lerp

Linear interpolation. Blends between two points. The third parameter (0‚Äì1) controls how fast the camera catches up.

‚è± Time.deltaTime

Multiplying by Time.deltaTime makes the smoothing frame-rate independent. 10 * deltaTime ‚âà quick but smooth follow.

üéö Tuning the Speed

Lower values (2‚Äì5) = lazy, cinematic feel. Higher values (10‚Äì20) = tight, responsive follow. At ~60fps, 10 * deltaTime ‚âà 0.17 per frame.

Complete Script

The Final CameraController

CameraController.cs ‚Äî Final Version 
using UnityEngine;

public class CameraController : MonoBehaviour
{
    public Transform target;    // Drag the Player here in Inspector
    private Vector3 offset;     // Auto-calculated at startup

    void Start()
    {
        offset = transform.position - target.position;
    }

    void LateUpdate()
    {
        Vector3 newPosition = new Vector3(
            transform.position.x,            // X: keep camera's X
            transform.position.y,            // Y: keep camera's Y
            offset.z + target.position.z     // Z: follow player + offset
        );

        transform.position = Vector3.Lerp(
            transform.position,
            newPosition,
            10 * Time.deltaTime
        );
    }
}
üîå Don't Forget!

After saving, go back to Unity, select the Main Camera, and drag the Player game object into the Target slot in the Inspector. Without this, the script won't know which object to follow.

Summary

Key Takeaways

üìê

Offset = Camera ‚àí Player

Capture the initial gap in Start(). This single calculation drives the entire follow system.

üîí

Lock Axes You Don't Need

Keep X and Y from the camera's own position. Only update Z to follow the player forward.

‚è±

LateUpdate for Cameras

Always put camera logic in LateUpdate() so it runs after the player has finished moving.

üßà

Lerp for Smoothness

Vector3.Lerp with Time.deltaTime creates frame-rate independent smooth following.

üîå

Wire Up in Inspector

Public Transform target means you must drag the Player into the slot. No assignment = NullReferenceError.

üéö

Tune the Follow Speed

The Lerp multiplier (currently 10) controls tightness. Lower = cinematic lag, higher = snappy tracking.
