01

FIX MOVEMENT BUG

Resolving CharacterController collision issues

Understanding the Problem

Before implementing game over, we need to fix a critical bug in our player movement. Currently, we're using two different methods to move the player: CharacterController.Move() for forward movement and transform.position for lane switching. This inconsistency causes the CharacterController's collider to become misaligned, allowing the player to pass through obstacles!

Critical Bug

Using transform.position directly with CharacterController causes collision detection to fail. The player will pass through obstacles like traffic cones!

Solution 1: Quick Fix (Controller Center)

The simplest fix is to manually update the controller's center position after changing the transform. This forces the CharacterController to recalculate its collider bounds:

PlayerController.cs 
// After setting transform.position, add this line:
controller.center = controller.center;

// This triggers the CharacterController to recalculate its bounds

Solution 2: Proper Fix (Use Controller.Move)

The better solution is to use CharacterController.Move() for all movement, including lane switching. This keeps everything consistent and ensures proper collision detection:

PlayerController.cs - Lane Movement Fix 
// Instead of: transform.position = targetPosition;
// Use this approach:

if (transform.position != targetPosition)
{
    // Calculate the difference between current and target
    Vector3 diff = targetPosition - transform.position;
    
    // Move using CharacterController
    controller.Move(diff);
}

Bonus Benefit

This approach also adds smoothness to lane switching movement, making the game feel more polished!

PLAYER MANAGER SCRIPT

Creating centralized game state management

Why a Separate Manager?

Instead of putting game state logic in the PlayerController, we create a dedicated PlayerManager script. This follows good software design principles by separating concerns: player movement in one script, game state in another. This manager will handle the game over state and control the UI panel visibility.

Create the Script and GameObject

  1. In the Project window, right-click â†’ Create â†’ C# Script
  2. Name it "PlayerManager"
  3. In the Hierarchy, create an empty GameObject
  4. Rename it to "PlayerManager"
  5. Drag and drop the script onto this GameObject

The PlayerManager Script

PlayerManager.cs 
using UnityEngine;

public class PlayerManager : MonoBehaviour
{
    // Static variable accessible from other scripts
    public static bool gameOver = false;
    
    // Reference to the Game Over UI panel
    public GameObject gameOverPanel;

    void Update()
    {
        // Check if game over state is triggered
        if (gameOver)
        {
            // Stop the game time
            Time.timeScale = 0;
            
            // Show the game over panel
            gameOverPanel.SetActive(true);
        }
    }
}

Understanding the Code

public static bool gameOver â€” The static keyword makes this variable globally accessible. Any script can read or change this value using PlayerManager.gameOver without needing a reference to the PlayerManager object.

Time.timeScale = 0 â€” This pauses all time-dependent operations in Unity. Physics, animations, and any time-based movement will freeze. This effectively pauses the game.

gameOverPanel.SetActive(true) â€” This enables the previously hidden UI panel, making it visible to the player.

Important Setup

After creating the script, select the PlayerManager GameObject in the Hierarchy and drag the GameOverPanel into the "Game Over Panel" field in the Inspector!

OBSTACLE TAGGING

Identifying obstacles with Unity tags

Why Use Tags?

When the player collides with something, we need to know what they collided with. Was it an obstacle? The road? A coin? Tags let us categorize GameObjects so we can respond differently to each type of collision.

Create the Obstacle Tag

  1. Select any GameObject in the scene
  2. In the Inspector, click the Tag dropdown (shows "Untagged")
  3. Click "Add Tag..."
  4. Click the + button under Tags
  5. Type "Obstacle" and press Enter/Save

Apply Tags to All Obstacles

Now go through your tile prefabs and tag every obstacle:

  1. Open the Prefabs folder in the Project window
  2. Double-click each tile prefab to open it
  3. Select each obstacle (traffic cones, roadworks barriers, etc.)
  4. In the Inspector, change the Tag from "Untagged" to "Obstacle"
  5. Save the prefab and repeat for all tiles

Prefab Tip

Double-click a prefab to open it in isolation mode. Any changes you make will apply to all instances of that prefab in your game!

Obstacles to Tag

Make sure to tag these common obstacle types in your game:

  • Traffic Cones â€” Both single and grouped variants
  • Roadworks Barriers â€” Construction barriers
  • Other hazards â€” Any object the player should avoid

COLLISION DETECTION

Detecting when player hits obstacles

OnControllerColliderHit Method

Since we're using a CharacterController (not a Rigidbody), we use Unity's special collision callback method: OnControllerColliderHit. This method is automatically called whenever the CharacterController collides with another collider.

Add to PlayerController Script

Open your PlayerController script and add this method (it goes outside of Start and Update, like any other method):

PlayerController.cs - Collision Detection 
// Called when CharacterController hits a collider
void OnControllerColliderHit(ControllerColliderHit hit)
{
    // Check if we hit an obstacle
    if (hit.transform.tag == "Obstacle")
    {
        // Trigger game over state
        PlayerManager.gameOver = true;
    }
}

Understanding the Code

ControllerColliderHit hit â€” This parameter contains all the information about what we collided with: the transform, collider, point of contact, and more.

hit.transform.tag â€” We access the tag of the object we hit. If it matches "Obstacle", we know the player ran into something they shouldn't have!

PlayerManager.gameOver = true â€” We set the static variable to true. The PlayerManager's Update method will detect this change and trigger the game over sequence.

BUTTON EVENTS

Wiring up replay and quit functionality

Create the Events Script

We need a script to handle button click events. Create a new C# script called "Events" and attach it to an empty GameObject also named "Events".

Events.cs 
using UnityEngine;
using UnityEngine.SceneManagement;

public class Events : MonoBehaviour
{
    // Called when Replay button is clicked
    public void ReplayGame()
    {
        // Reset time scale (was set to 0 on game over)
        Time.timeScale = 1;
        
        // Reset game over state
        PlayerManager.gameOver = false;
        
        // Reload the current scene
        SceneManager.LoadScene("Level");
    }
    
    // Called when Quit button is clicked
    public void QuitGame()
    {
        // Exit the application
        Application.Quit();
    }
}

Understanding the Code

using UnityEngine.SceneManagement â€” This namespace is required to use scene loading functions.

Time.timeScale = 1 â€” We reset time to normal speed before reloading. If we don't do this, the new scene will also be paused!

SceneManager.LoadScene("Level") â€” This reloads the scene by name. Make sure "Level" matches your actual scene name exactly.

Application.Quit() â€” Exits the game. Note: This only works in a built game, not in the Unity Editor.

Critical: Reset Time Scale

Always reset Time.timeScale to 1 before loading a new scene! Otherwise, your reloaded game will be frozen because timeScale is still 0.

Connect Buttons to Events

Now we wire up the UI buttons to call these methods:

  1. Select the ReplayButton in the Hierarchy
  2. In the Inspector, find the OnClick() section
  3. Click the + button to add an event
  4. Drag the Events GameObject into the object field
  5. Click the dropdown and select Events â†’ ReplayGame

Repeat the same process for the QuitButton, but select Events â†’ QuitGame instead.

Test the Complete System

  1. Make sure the GameOverPanel is deactivated
  2. Press Play in Unity
  3. Move around until you hit an obstacle
  4. The game should freeze and show the Game Over panel
  5. Click Replay to restart the game
  6. The game reloads and you can play again!
