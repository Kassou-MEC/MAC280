Creating the Game Over UI

Setting up the game over screen with text and button

UI Hierarchy Setup

Create a proper UI hierarchy in Unity:

  • Create an empty GameObject called "Game Over Screen" under Canvas
  • Add a Text element as a child for "Game Over" message
  • Add a Button element (under Legacy) as another child
  • Resize and position elements appropriately
  • Find and modify the Text child of the button to change button text

Key Unity UI Concepts

  • Canvas is the root container for all UI elements
  • UI elements use Rect Transform instead of regular Transform
  • Anchors determine how UI elements position relative to canvas
  • Buttons have built-in interactivity and visual states
  • Text elements display information to the player

Button Event System

Unity's button component has an "On Click" event:

  • Allows calling public functions on GameObjects
  • Can be configured directly in the Inspector
  • Supports multiple function calls on a single click
  • Functions must be public to appear in the event list

RestartGame Function Code

using UnityEngine.SceneManagement;

public void RestartGame()
{
    // Reload the current scene
    SceneManager.LoadScene(SceneManager.GetActiveScene().name);
}

Add this to your logic script, then connect it to the button's OnClick event in Unity.

Creating the Game Over Function

Making the game over screen appear when the bird crashes

GameOver Function

Create a public function in the logic script:

public GameObject gameOverScreen;

public void GameOver()
{
    // Show the game over screen
    gameOverScreen.SetActive(true);
}
  • Make a reference to the game over screen GameObject
  • Fill this reference in Unity's Inspector
  • Call SetActive(true) to show the screen

Initial Setup

Before the game starts:

  • Disable the game over screen GameObject using the checkmark in Unity
  • This ensures it's hidden at the start of the game
  • It will only appear when GameOver() is called
  • This is better than moving it on/off screen

Detecting Bird Collisions

Using OnCollisionEnter2D to detect when the bird hits pipes

Clip 1

Clip 1

Collision Detection Code

private LogicScript logic;

void Start()
{
    logic = GameObject.FindGameObjectWithTag("Logic").GetComponent<LogicScript>();
}

private void OnCollisionEnter2D(Collision2D collision)
{
    logic.GameOver();
}

Add this to your bird script to detect collisions with pipes.

Trigger vs Collision

  • OnTriggerEnter2D: Used when objects pass through each other (like scoring zones)
  • OnCollisionEnter2D: Used when objects are solid and should collide (like pipes)
  • Pipes should have Colliders (not set as triggers)
  • Bird needs a Collider to detect collisions
  • Both need Rigidbody2D components for physics

Managing Game State with Booleans

Using a boolean variable to track if the bird is alive

Boolean Variables

Boolean (bool) variables are simple true/false values:

  • true = on, yes, alive
  • false = off, no, dead
  • Perfect for tracking game states
  • Great for conditional checks in code
public bool birdIsAlive = true;

public void GameOver()
{
    gameOverScreen.SetActive(true);
    birdIsAlive = false;
}

Updated Input Check

Modify the bird's input check to include the boolean:

void Update()
{
    if (Input.GetKeyDown(KeyCode.Space) && birdIsAlive)
    {
        // Make the bird flap
        myRigidbody.velocity = Vector2.up * flapStrength;
    }
}

The bird will only flap if both conditions are true: space is pressed AND birdIsAlive is true.

Building Your Game

To create a standalone version of your game:

  • Go to File > Build Settings
  • Make sure your scene is in the "Scenes In Build" list
  • Choose your target platform (Windows, Mac, etc.)
  • Click Build and select a folder
  • Unity will compile your game into an executable

Modern Unity Alternatives

The tutorial uses simpler, older systems:

  • New Input System - Better than Input.GetKeyDown
  • TextMeshPro - Replaces legacy UI text
  • UI Toolkit - Newer UI system (still evolving)
  • These are more complex but more powerful
  • Learn basics first, then upgrade later
