1

Create UI Text

Setting up the "Tap to Start" text element

Creating the Text Object

First, we need to add a UI Text element to display the "Tap to Start" message. This text will be visible when the game loads and will have an animated effect to catch the player's attention.

  1. In the Hierarchy, right-click â†’ UI â†’ Text
  2. Rename the text object to "StartingText"
  3. Position it in the center of the screen (or wherever you prefer)

Configuring Text Properties

Customize the text appearance to make it stand out and look professional:

  1. Set Width to 150 and Height to 60
  2. Change Font Size to 30
  3. Set Font Style to Bold
  4. Change the text content to "TAP TO START"
  5. Set Alignment to center (both horizontal and vertical)
  6. Choose a Color that stands out (the tutorial uses a bright color)

Design Tip

Choose a color that contrasts well with your game background. Bright colors like yellow, cyan, or white work great for visibility!

2

Create Scale Animation

Adding a pulsing effect to the text

Why Animate the Text?

A static "Tap to Start" message can be easily overlooked. By adding a subtle pulsing animation, we draw the player's attention and create a more polished, professional feel. The animation we'll create scales the text up and down rhythmically.

Opening the Animation Window

  1. Select the StartingText object in the Hierarchy
  2. Go to Window â†’ Animation â†’ Animation
  3. Dock the Animation window where you can see both it and the text
  4. Click the Create button in the Animation window
  5. Create a new folder called "Animations"
  6. Save the animation as "TextAnim"

Creating Keyframes

Now we'll set up the scale animation using keyframes. The text will start at normal size, grow larger, then return to normal size in a loop:

  1. Click the Record button (red circle) to start recording
  2. At Frame 0: Set Scale to (1, 1, 1) - normal size
  3. Move to Frame 30: Set Scale to (1.5, 1.5, 1) - enlarged
  4. Move to Frame 60: Set Scale back to (1, 1, 1) - normal size
  5. Click the Record button again to stop recording

Note About Z Scale

We don't need to change the Z scale because our text is 2D. Only X and Y scaling matter for UI elements.

Add Game State Logic

Preventing movement until tap

The Problem

Right now, the player starts moving as soon as the game loads. We need a way to pause the game until the player taps the screen. This requires a state variable that tracks whether the game has started.

Modify PlayerManager Script

Open your PlayerManager.cs script and add a new static boolean variable:

PlayerManager.cs 
public class PlayerManager : MonoBehaviour
{
    // Existing game over variable
    public static bool gameOver = false;
    
    // NEW: Track if the game has started
    public static bool isGameStarted = false;
    
    // Reference to the UI panel
    public GameObject gameOverPanel;
    
    // ... rest of your code
}

Block Player Movement

Now we need to modify the PlayerController.cs to check this variable before allowing movement:

PlayerController.cs - Update Method 
void Update()
{
    // If game hasn't started, don't process any movement
    if (!PlayerManager.isGameStarted)
    {
        return;  // Exit the method early
    }
    
    // Your existing movement code goes here...
}

Also Block FixedUpdate

If you're using FixedUpdate() for physics-based movement, add the same check there:

PlayerController.cs - FixedUpdate Method 
void FixedUpdate()
{
    // If game hasn't started, don't move
    if (!PlayerManager.isGameStarted)
    {
        return;
    }
    
    // Your existing physics movement code...
}

Result

Now when you play the game, the player will stay still until isGameStarted becomes true!

4

Handle Tap Input & Cleanup

Starting the game and removing the text

Detecting the Tap

We need to detect when the player taps the screen and then start the game. Add this logic to your PlayerManager.cs Update method:

PlayerManager.cs - Tap Detection 
void Update()
{
    // Check for tap input using SwipeManager
    if (SwipeManager.tap)
    {
        // Start the game!
        isGameStarted = true;
    }
    
    // Existing game over check...
    if (gameOver)
    {
        Time.timeScale = 0;
        gameOverPanel.SetActive(true);
    }
}

Destroying the Start Text

Once the player taps, we need to remove the "Tap to Start" text from the screen. First, add a reference to the text object, then destroy it when tapped:

PlayerManager.cs - Complete Implementation 
using UnityEngine;

public class PlayerManager : MonoBehaviour
{
    public static bool gameOver = false;
    public static bool isGameStarted = false;
    
    public GameObject gameOverPanel;
    
    // Reference to the starting text
    public GameObject startingText;

    void Update()
    {
        // Check for tap to start the game
        if (SwipeManager.tap)
        {
            // Start the game
            isGameStarted = true;
            
            // Remove the starting text
            Destroy(startingText);
        }
        
        // Handle game over state
        if (gameOver)
        {
            Time.timeScale = 0;
            gameOverPanel.SetActive(true);
        }
    }
}

Don't Forget!

In Unity, select the PlayerManager GameObject and drag your StartingText into the "Starting Text" field in the Inspector!

Test Your Implementation

  1. Press Play in Unity
  2. Verify the "Tap to Start" text is visible and animating
  3. Verify the player is NOT moving
  4. Tap/click the screen
  5. The text should disappear and the player should start moving
  6. Swipe controls should now work as expected
